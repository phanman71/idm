Creative Coding Week 5

For this week in creative coding, I learned about APIs and how to use them.

Looking through the list of free API's I decided to go with the game list from Giant Bomb since I like video games alot. This one was a little tricky to use, first to gain access to it, I had to make an account and get an access key. To get it working I also had to setup a COR. This part was pretty tricky because after a while I found out I had to set it up as a proxy.

The Giant Bomb API uses a 5 digit code for each of it's games so I decided to set up a random game generator. Going through the options for available keywords, I found "images" was a nice to grab an image of the game. So after that I setup a function where when you click, you'll get a new game image along with it's name and ID. Sometimes you'll get a number that is not registered to a game resulting in nothing happening. I did try to fix this but haven't quite figured it out yet. So whenever you do get a new image, it goes next to a previous one or below creating a scroll bar, I'm not completely sure how this works out or how to scale the images but I think I'm pretty happy with the outcome. With so many video games out there, I'm suprised that I know some of the ones I've had pop up.

Art and the API

I always knew data was really valueable for companies but I never really thought of exactly what that data was. In a way API are kinda like classes in where you can take information out of them or how it's owners can put info in which makes it perfect to use with object oriented programming.
